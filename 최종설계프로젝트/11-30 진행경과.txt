노드레드 대시보드의 접속 경로를 QR코드로 생성
-> 모바일 기기로 QR 인증하면 대시보드로 이동
 -> 버튼을 눌러 사진촬영 =IP CAM 어플(백그라운드에서 동작하기 옵션)과 서버 사용 (해당서버의 IP 주소는 노드레드 에디터 상에 미리 전처리를 해줘야함)
  -> 사진 촬영 후 결과를 모바일 기기 상에서 확인 가능 (모자 착용시 Helmet or None/ 미착용시 무조건 None)

< 보완 > 
1. IP CAM 경로를 노드레드 에디터 상에서 받을 수 있을지?
2. Helmet 인증이 안된 경우 횟수를 부여
3. 결과 출력시 사운드를 "출력 헬멧을 착용했습니다. 붕붕이가 시작됩니다!"  , "헬멧이 확인되지 않습니다. 다시 시도해주십시오."
4. 화면에 사용자의 카메라 상황이 띄워지면 좋을 것 같다. 

추가: 민지가 말한 붕붕이의 상황 여부를 출력하는 경우 터치 센서를 활용하면 "사용중 -> 정지 -> 다시 가동" 을 보여주기 편할 것 같음


ex: 터치 둘다 잡으면 : 액셀 밟은 상황 -> 하나라도 안 잡히면 멈춘것으로 인식 : 정지한 상황
